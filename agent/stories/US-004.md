## Título Ingreso de datos de remitente y destinatario

COMO:     Usuario que envía remesas internacionales
QUIERO: Mantener historial de datos
PARA: Facilitar/agilizar futuros trámites de envíos de divisas

```gherkin
Feature: Ingreso de datos de remitente y destinatario
  As a usuario que envía remesas internacionales
  I want mantener historial de datos
  So that facilitar/agilizar futuros trámites de envíos de divisas

  Scenario 1: Guardado automático de datos del remitente
    Given el usuario completa el formulario de envío por primera vez con sus datos de remitente (nombre, identificación, número de cuenta, dirección, teléfono y país)
    When confirma el envío o guarda la información
    Then el sistema almacena estos datos en el backend local para reutilizarlos en futuros envíos

  Scenario 2: Guardado automático de datos del destinatario
    Given el usuario introduce los datos del destinatario en el formulario (nombre, identificación, número de cuenta, dirección, teléfono y país)
    When confirma el envío o guarda la información
    Then el sistema guarda estos datos en el backend local como destinatario frecuente

  Scenario 3: Auto llenado del remitente en futuros envíos
    Given el usuario ya tiene datos de remitente guardados
    When accede nuevamente al formulario de envío
    Then los campos del remitente aparecen autocompletados automáticamente, permitiendo editar cualquier dato antes de enviar

  Scenario 4: Selección de destinatario frecuente
    Given el usuario tiene varios destinatarios guardados
    When abre el formulario de envío
    Then puede seleccionar un destinatario desde una lista y los datos se autocompletan automáticamente en el formulario

  Scenario 5: Edición de datos guardados
    Given el usuario accede a la sección de gestión de perfiles
    When modifica los datos de un remitente o destinatario y guarda los cambios
    Then el sistema actualiza la información almacenada en el backend local

  Scenario 6: Eliminación de un destinatario frecuente
    Given el usuario tiene destinatarios guardados
    When selecciona la opción "Eliminar" en uno de ellos
    Then el sistema solicita confirmación y, si el usuario acepta, elimina el destinatario del backend local

  Scenario 7: Persistencia de datos
    Given el usuario ha guardado datos previamente
    When vuelve a abrir la aplicación
    Then los datos de remitente y destinatarios se cargan automáticamente desde el backend local sin necesidad de volver a escribirlos

  Scenario 8: Manejo de errores
    Given el sistema intenta cargar los datos guardados
    When ocurre un error en la lectura del backend local
    Then se muestra un mensaje claro de error y se ofrece la opción de reintentar o continuar sin autollenado
```
